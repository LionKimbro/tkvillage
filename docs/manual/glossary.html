Glossary

Glossary

Short meanings for the recurring TkVillage words.

TermMeaning
appThe single TkVillage runtime for the process.
rt.gThe runtime dictionary for scalar-ish values and handles such as root, project_dir, shutdown policy, and tick count.
rootThe hidden tkinter.Tk object that anchors Tkinter.
ToplevelA visible Tkinter application window.
window_kindAn app-chosen name for a registered window recipe.
window_idA TkVillage-generated id for one live window, currently shaped like window-1.
instance_keyA reuse key derived from a window kind and summon policy.
target_idAn event/message routing address. For windows, this equals the window id.
targetAnything with an inbox that can receive queued input. Windows and services are targets.
service targetA non-window target registered with register_service_target().
eventA semantic dictionary queued for a reducer or service handler.
messageA payload wrapped in a MESSAGE event envelope.
effectA semantic dictionary returned by a reducer or handler for the runtime to route.
reducerA function that turns state, event into new_state, effects.
projectionThe function that updates widgets from reducer-owned state.
tickOne runtime heartbeat: drain queues, reduce events, route effects, project windows, check shutdown.
shutdown policyThe declared rule for when TkVillage begins graceful shutdown.
standalone modeTkVillage owns its own project directory declaration and readiness.
lionscliapp modelionscliapp hosts the command-line app and resolved project directory; TkVillage becomes ready inside commands.
