Testing and Debugging

The tick loop is explicit enough to test. The debug snapshot is JSON-friendly enough to hand to humans or LLMs.

Manual Ticks

village.declare_app({
    "name": "test-app",
    "project-dir-name": ".test-app",
    "shutdown-policy": "explicit",
    "test-mode": True,
    "create-root": False,
})

village.tick_once()
village.run_ticks(3, update_tk=False)

Debug Snapshot

snapshot = village.get_debug_snapshot()
text = village.copy_debug_snapshot()

The snapshot includes runtime name, tick count, registered window kinds, live windows, recent events, recent effects, messages, and runtime logs.

In test-mode, tick exceptions are re-raised after logging, which makes failing tests direct instead of silent.
