Shutdown and Config

TkVillage makes application exit policy explicit and keeps simple configuration in the project directory.

Shutdown Policy

PolicyMeaning
on-last-window-closeAfter at least one window has existed, shutdown begins at tick end when no live windows remain.
on-window-closeAfter a specific window kind has existed, shutdown begins at tick end when no live windows of that kind remain.
explicitThe app keeps running without windows until request_shutdown(reason=None) is called.
def cleanup():
    print("closing cleanly")

village.declare_app({
    "name": "my-app",
    "project-dir-name": ".my-app",
    "shutdown-policy": "explicit",
    "shutdown-window-kind": None,
    "on-shutdown": cleanup,
})

village.request_shutdown("user requested exit")

Configuration

Config declarations describe type and intent. Values are saved to config.json under rt.g["project_dir"].

village.declare_config({
    "name": "theme",
    "default": "light",
    "type": "choice",
    "description": "UI theme",
    "choices": ["light", "dark"],
})
village.set_config("theme", "dark")
theme = village.get_config("theme")
