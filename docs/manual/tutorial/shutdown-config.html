Shutdown and Hosting

TkVillage makes application exit policy explicit and keeps its own runtime files in a small project directory.

Shutdown Policy

PolicyMeaning
on-last-window-closeAfter at least one window has existed, shutdown begins at tick end when no live windows remain.
on-window-closeAfter a specific window kind has existed, shutdown begins at tick end when no live windows of that kind remain.
explicitThe app keeps running without windows until request_shutdown(reason=None) is called.
def cleanup():
    print("closing cleanly")

village.declare_app({
    "name": "my-app",
    "project-dir-name": ".my-app",
    "shutdown-policy": "explicit",
    "shutdown-window-kind": None,
    "on-shutdown": cleanup,
})

village.request_shutdown("user requested exit")

Hosting

Most TkVillage apps are expected to run inside lionscliapp. In that mode, declaration does not create files because lionscliapp resolves the real project directory later, inside command runtime.

village.declare_app({
    "name": "my-app",
    "lionscliapp": True,
    "shutdown-policy": "explicit",
    "shutdown-window-kind": None,
    "on-shutdown": cleanup,
})

def my_command():
    village.ensure_ready()
    village.run()

Standalone mode remains available for smaller scripts. Provide project-dir-name or project-dir; ensure_ready() creates that directory when the runtime enters use.

village.declare_app({
    "name": "small-tool",
    "project-dir-name": ".small-tool",
    "shutdown-policy": "on-last-window-close",
    "shutdown-window-kind": None,
    "on-shutdown": None,
})
