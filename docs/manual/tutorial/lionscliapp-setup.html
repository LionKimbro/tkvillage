lionscliapp Setup

An alternate way to start a TkVillage app: lionscliapp hosts the command-line application, and TkVillage runs the GUI.

You do not need lionscliapp to use TkVillage. The Quick Start path is enough for ordinary standalone Tkinter apps. Use this page when you want to use TkVillage with lionscliapp. lionscliapp is framework for building small (mostly) command-line tools. It provides a project folder, configurable data that can also be persisted, command discovery and execution.

1. Choose This Startup Path Intentionally

In standalone mode, TkVillage can declare its own project directory and start with village.run(). In lionscliapp-hosted mode, your the TkVillage code's main routine runs through app.main().

# Standalone shape:
village.declare_app({...})
village.summon_window("main")
village.run()

# lionscliapp-hosted shape:
app.declare_app(...)
app.declare_projectdir(...)
village.declare_app({"lionscliapp": True, ...})
app.declare_cmd("", main_command)
app.main()

2. Let lionscliapp Own the Project Directory

The tkvillage project folder sits within lionscliapp's project folder. Its existence is ensured once tkvillage.ensure_ready() is called at runtime.

def main_command():
    village.ensure_ready()  # This line creates the tkvillage/ project folder.
    register_windows()
    village.summon_window("main")
    village.run()

3. Declare lionscliapp Normally

import lionscliapp as app
import tkvillage as village

app.declare_app("myapp", "1.0")
app.declare_projectdir(".myapp")

4. Declare TkVillage in Hosted Mode

Use one TkVillage declaration dictionary. In hosted mode, omit project-dir-name; lionscliapp supplies the host project directory later.

village.declare_app({
    "name": "myapp",
    "lionscliapp": True,  # <----
    "shutdown-policy": "on-window-close",
    "shutdown-window-kind": "main",
    "on-shutdown": None,
})

5. Ensure Readiness Inside Commands

Call village.ensure_ready() at the top of commands that use TkVillage. At that point, lionscliapp has entered runtime and TkVillage can ask lionscliapp for the resolved project root.

def main_command():
    village.ensure_ready()
    register_windows()
    village.summon_window("main")
    village.run()

app.declare_cmd("", main_command)
app.main()

6. Resulting Layout

<execroot>/
  <project-dir>/
    config.json
    tkvillage/

Application configuration remains with lionscliapp. TkVillage-specific runtime files belong under the tkvillage/ subdirectory.

7. Avoid These Moves

  • Do not call ensure_ready() during module import.
  • Do not call ensure_ready() during declaration.
