declare_app

declare_app(spec)
module: tkvillage.appauthority: runtime declaration

Preferred application declaration API. It resets runtime state, records the hosting/project-directory mode, optionally creates the hidden root, registers debug windows, and installs the debug shortcut.

Parameters

NameTypeHook
specdictApplication declaration bundle using public kebab-case keys.

spec Bundle

NameTypeHook
namestrRequired runtime/application name stored in rt.g["name"].
project-dir-namestrStandalone project-local runtime directory name. Required unless project-dir is supplied or lionscliapp is true.
project-dirstr | pathlib.PathOptional explicit standalone project directory path.
lionscliappboolOptional host-mode flag. When true, TkVillage defers project path resolution until ensure_ready().
shutdown-policy"explicit" | "on-last-window-close" | "on-window-close"Required policy for when graceful shutdown begins.
project-rootstr | pathlib.Path | NoneOptional parent directory under which the project directory is resolved.
tick-interval-msint | strOptional tick loop interval in milliseconds; coerced to int.
test-modeboolOptional flag causing tick exceptions to re-raise after logging.
create-rootboolOptional flag; when false, declaration does not create the Tk root.
shutdown-window-kindstr | NoneRequired when policy is on-window-close; otherwise usually None.
on-shutdowncallable | NoneOptional no-argument function called once when shutdown begins.

Returns

rt.g.

Things to Know

shutdown-policy is non-optional. In lionscliapp mode, call ensure_ready() at the top of commands that use TkVillage. Use this for new code; create_app() is deprecated.

request_shutdown

request_shutdown(reason=None)
authority: graceful shutdown request

Sets the explicit shutdown flag and optional reason. Shutdown begins during the tick-end policy check.

Parameters

NameTypeHook
reasonany JSON-friendly value | NoneOptional explanation stored in rt.g["shutdown_reason"] and written to the runtime log.

Side Effects

Updates rt.g["shutdown_requested"], rt.g["shutdown_reason"], and appends to rt.runtime_log.

reset_app

reset_app()
authority: runtime reset

Resets runtime state to defaults while preserving dictionary and list identities.

Things to Know

Destroys an existing root if possible. Uses clear() and update() rather than rebinding runtime containers.

ensure_ready

ensure_ready()
authority: runtime readiness

Creates and records the TkVillage project directory when the runtime is actually entering use. In lionscliapp mode, it asks lionscliapp.paths.get_project_root() for the host project directory, then creates <host-project>/tkvillage.

Returns

rt.g.

Things to Know

Call this at the top of lionscliapp commands that use TkVillage. run() calls it automatically.

run

run()
authority: Tk mainloop

Ensures runtime readiness, ensures the hidden root exists, starts the tick loop, and enters tkinter.mainloop().

Things to Know

This blocks until the Tk mainloop exits.

shutdown

shutdown()
authority: hard cleanup

Stops ticking, calls the shutdown handler once if configured, destroys live windows and root, and marks the runtime not running.
