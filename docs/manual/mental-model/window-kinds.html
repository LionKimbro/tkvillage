Glossary

Window Kinds

A window kind is a named recipe. It describes how future windows of that kind are created, reused, reduced, projected, and closed.

What a Kind Contains

FieldPurpose
window_kindStable symbolic name.
multiplicityHow summon behaves: singleton, per-key, or free-instance.
make_initial_stateCreates reducer-owned semantic state.
createBuilds widgets into a live record.
reduce_eventTransforms state using semantic events.
projectUpdates widgets from state.
on_show/on_close/on_destroy/on_tickLifecycle hooks.

Multiplicity

singleton

One live instance for the kind. Summon raises it if it already exists.

per-key

One live instance per key. Good for editors, inspectors, or object detail windows.

free-instance

Every summon creates a fresh independent window.

embedded and modal are recognized names but currently raise NotImplementedError.
