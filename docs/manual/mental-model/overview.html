Mental Model

TkVillage is a little operating system for Tkinter windows. It keeps runtime facts in one place, lets windows receive queued meaning, and advances the application through ticks.

Runtime

The shared host organism. It owns the hidden root, registries, queues, logs, and tick settings.

Window Kind

A registered recipe: title, multiplicity, create function, reducer, projection, and hooks.

Window

A live Toplevel with a record: id, kind, state, event queue, widgets, lifecycle flags.

Target

Anything with an inbox. Windows are targets. Services can be targets too.

The Main Loop of Meaning

Tk callback translates raw UI activity into a semantic event.
Event enters a window or service inbox.
Tick drains queues and calls reducers or handlers.
Reducers return new state plus explicit effects.
Effects are routed; dirty windows are projected from state to widgets.
The core bargain: Tk callbacks translate, reducers decide, projections display.
