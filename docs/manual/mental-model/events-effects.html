Glossary

Events and Effects

Events are meanings waiting to be reduced. Effects are explicit requests for runtime work outside the reducer.

Event Shape

{"type": "SAVE_REQUESTED", "path": "notes.json"}

Events should be JSON-friendly where practical. They describe what happened in application terms, not raw Tk details.

Effect Shape

{"type": "POST_EVENT", "target_id": "window-2", "event": {"type": "REFRESH"}}

Reducers return effects when they need the runtime to do work: post another event, send a message, summon a window, close a window, request projection, log, or show a message box.

EffectWhat it asks the runtime to do
POST_EVENTQueue an event to a target.
SEND_MESSAGEWrap and queue a message.
SUMMON_WINDOWCreate or raise a window.
CLOSE_WINDOWClose a window through its close hook.
DESTROY_WINDOWDestroy a window directly.
REQUEST_PROJECTMark a window dirty.
LOGWrite to the runtime log.
SHOW_MESSAGE_BOXShow a simple Tk message box.
Events enter reducers. Effects leave reducers.
