Tick Loop

The tick is TkVillage's heartbeat. It turns queued input into state changes, routes effects, projects windows, and checks whether shutdown should begin.

Increment rt.g["tick_count"].
Drain service target inboxes by calling handler(target, event).
Drain window event queues by calling reduce_event(state, event).
Route returned effects.
Call each window on_tick(record) hook.
Project dirty windows with project(record).
Evaluate shutdown policy at tick end.
A reducer should not perform long-running work. Use queued services or future worker facilities for work that should not block the UI.
